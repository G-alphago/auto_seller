# 큐텐 재팬 자동 업로드 파이프라인 작업 보고서 (qoo10walkthrough)

기존 시스템의 버그를 수정하고, 대량 업로드를 위한 다중 URL 처리 및 결과물 분리 기능을 완벽히 구현하였습니다.

## 1. 주요 구현 및 수정 사항

### 🛠 데이터 추출 및 필터링 (scraper.py)
*   **버그 수정**: 상세 페이지 본문이 스킵되던 원인인 `layout`, `commonLayout` 필터를 제거하여 모든 본문 데이터를 정상 추출하도록 개선했습니다.
*   **환경 최적화**: 서버/에이전트 환경에서 원활한 실행을 위해 Playwright를 `headless=True` 모드로 설정하고, 임시 폴더 권한 문제를 해결하는 로직을 적용했습니다.

### 🌐 자동 번역 및 HTML 보존 (converter.py)
*   **HTML 번역**: `BeautifulSoup`을 사용하여 상세 설명의 HTML 태그 구조는 그대로 유지하면서 내부에 포함된 한글 텍스트만 일본어로 번역하는 기능을 추가했습니다.
*   **이미지 규격화**: 메인 이미지를 중앙 정렬하고 흰색 배경으로 패딩 처리하여 큐텐 가이드(800x800 정사각형)에 맞게 자동 변환 후 `outputs/images/`에 저장합니다.

### 📊 엑셀 출력 및 결과 분리 (exporter.py)
*   **다중 행 지원**: 여러 상품 데이터를 한 번에 입력받아 하나의 업로드용 엑셀 파일로 통합 생성하도록 기능을 확장했습니다.
*   **요약 파일 생성**: 원본 URL과 주요 정보를 매칭하여 관리할 수 있는 `summary_날짜.xlsx` 별도 생성 기능을 추가했습니다.

### 💻 사용자 인터페이스 (Web UI & Batch)
*   **웹 UI 개편**: `index.html`을 수정하여 **10개의 URL 입력 칸**을 제공하며, 한 번에 2종의 엑셀 파일을 모두 다운로드할 수 있도록 했습니다.
*   **배치 스크립트**: 대량 처리를 위한 전용 파일(`test_batch.py`)을 생성하여 수십 개의 URL도 터미널에서 즉시 처리 가능하게 했습니다.

## 2. 프로젝트 실행 방법

### 환경 설정 (최초 1회)
```bash
# 필수 라이브러리 설치
python3 -m pip install -r requirements.txt
python3 -m pip install Pillow playwright deep-translator
python3 -m playwright install
```

### 방식 A: 웹 화면에서 사용 (추천)
1.  터미널에서 `python3 app.py` 실행
2.  브라우저에서 `http://127.0.0.1:8080` 접속
3.  10개의 입력창에 URL 입력 후 '실행' 버튼 클릭

### 방식 B: 터미널에서 배치 처리
1.  `test_batch.py` 하단의 `test_urls` 리스트에 URL 추가
2.  터미널에서 명령 실행: `python3 test_batch.py`

### 📁 프로젝트 구조 최적화 (dev_tools 도입)
*   **환경 정제**: 루트 폴더에 산재해 있던 20여 개의 테스트(`test_*.py`) 및 디버깅(`debug_*.py`) 스크립트들을 `dev_tools/` 폴더로 일괄 이동하여 작업 환경을 깔끔하게 정리했습니다.
*   **핵심 파일 배치**: `app.py`, `scraper.py`, `converter.py` 등 실제 서비스 구동에 필요한 파일들만 루트에 노출시켜 직관성을 높였습니다.

### 🖼️ 이미지 URL 로직 원복 및 호환성 확보 (converter.py)
*   **버그 해결**: 최근 추가된 '이미지 가공' 기능으로 인해 엑셀에 로컬 파일 경로가 기입되던 문제를 해결했습니다.

## 8. 판매자 상품 코드 기반 연동 체계 구축 및 모니터링 고도화 (2026-04-06)
데이터 결손 방지와 모니터링 시스템의 정확도 100% 달성을 위해 판매자상품코드 기반 파이프라인을 구축했습니다.

### 🔑 판매자상품코드(Seller Unique Item ID) 자동화 (exporter.py)
*   **코드 자동 생성**: 큐텐 업로드 엑셀 파일 생성 시, 파일명의 뒷자리 9자리와 일련번호를 조합해 B열에 자동으로 부여합니다 (예: `03_123629_01`).
*   **단일화된 관리망 구비**: 새롭게 생성된 모든 서머리(`summary_*.xlsx`) 파일의 A열에도 식별자가 통합 동기화되게 수정하여 관리 편의성이 극대화되었습니다.
*   **단발성 마이그레이션**: `outputs/`에 남아있던 수십 개의 구버전 업로드/서머리 엑셀들에 대해서도 판매자 코드를 소급 일괄 적용하는 스크립트를 즉석 가동해 동기화했습니다.

### 📊 모니터링 로직 근본적 개편 (monitor_stock.py)
*   **텍스트 비교 제거 및 직통 매칭입력**: 기존처럼 상품명을 정규화하여 텍스트로 비교하는 방식을 전면 철거했습니다. 서머리의 A열에 위치한 **판매자상품코드**를 원본 데이터(큐텐업로드엑셀)와 직접 딕셔너리로 1:1 대조합니다.
*   **기준점 변경**: 웹 스크래퍼에서 긁어온 정보를 대조할 기준 데이터(Old price/options)를 서머리 파일이 아닌 **실제 원본 큐텐 업로드 파일 (`qoo10_upload_*.xlsx`)의 값으로 로딩**하도록 수정해, 픽셀 단위로 신뢰성을 보완했습니다.
*   **Qoo10_Edit 방식 최적화**: 모니터링 후 가격/재고 변경 내역이 모인 엑셀 파일 산출 시, B열에 판매자상품코드를 정확한 위치에 안착시키며 A열(아이템코드) 추가 연동 기능을 병합했습니다. 이를 통해 즉석에서 재고수정 업로드가 가능한 무결점 엑셀 결과물이 배출됩니다.
